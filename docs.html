The Board

Basics

When Rosewood is initialized, it creates an html div with the id 'board'. The board serves as the DOM parentNode for all backgrounds, sprites and invisible barriers within the game.

Entities

Entity Basics

Entities (hereforth know as 'ents') form the core of the Rosewood engine. Every non-background object that is append to the game board is initialized as an ent. All sprites, Characters, Enemies, Items and any invisble barriers or specific areas within the board are created using the rw.newEnt function and tracked within the rw.ents array.

A large portion of game design with Rosewood is creating custom ent definitions for the objects witin the game. however there are a number of properties and functions that all ents share.

ent.base and rw.ent

Every ent is created with a custom constructor function written for the specific game, but every ent constructor must have certain properties. In order to be considered valid by the engine an ent must possess a property named 'base' that references an rw.ent object. In other words, an ent named 'ent' would have a property 'ent.base' that would refer to an rw.ent object. The rw.ent object, by way of ent.base contains much of the information that all ents share. Properties within rw.ent are read by the engine to determine the dimensions and location of each entity as well as what sprites to represent them with onscreen. At a minimum, an ent MUST have an rw.ent object assigned to its 'base' property.

rw.ent properties

The rw.ent object contains the properties used by the engine for positioning and displaying entities onscreen. They are stored within an ent's 'base' property. The ent.base properties inherited from rw.ent are as follows:

  • ent.base.name

  • ent.base.typeClass

  • ent.base.sprites

  • ent.base.spritesSct

  • ent.base.width

  • ent.base.height

  • ent.base.posX, posY and posZ

  • ent.base.velX, velY and velZ

  • ent.base.heading

  • ent.base.moving

  • ent.base.active

  • ent.base.display()

  • ent.base.hide()

  • ent.base.changeSprite(spriteName)

ent.update() and ent.iGotHit()

While not necessary for validation, ents must have an 'update()' function if they are to be moved onscreen after initial placement. The engin runs at a specified rate of frames per second; in each frame, the update() function of every ent is called and if the ent moves, it is re-positioned onscreen. In addition to the update() function, ents can also have a function called 'iGotHit()' which contains instructions for how an entity should react if it collides with another entity. When two ents collide their 'ent.base.typeClass' value is passed to the other ent's iGotHit() funtion like so: ent.iGotHit(otherEnt.base.typeClass) This allows the game designer to specify different behavior for an ent running into a wall as opposed to say, the ent being hit by a bullet.

rw.ents Array

Rosewood stores every game entity (or 'ent') within the rw.ents array. When a new ent is created using the rw.newEnt() function it is appended to the end of the array. This means, that as ents are created and removed the array index of a given ent may change, making it difficult to refer to a specific ent using its location within the array. However, this shortcoming is easily mitigated by crafting an ent's properties and functions in a way that does not rely on an absolute index within the rw.ents array. In addition, the rules object provides methods of directly interfacing with specific ents. See the Rules section for more details.

Tutorial

Getting Started

Set Up

To start creating a game with Rosewood we first need to grab a copy of the engine and set up a project directory. Create a folder for the game and place a copy of rosewood.js to it. Then, using your favorite text-editor, create a file called 'mygame.js' and another called 'game.html'.
