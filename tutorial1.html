Intro

   For the longest time if you wanted to create a browser-based video game you had one choice, Adobe's Flash platform. Flash works quite well for making games but their are drawbacks, namely the cost of purchasing Adobe software and the closed-nature of the platform. With the advent of faster scripting engines in modern browsers it is now becoming possible to create games using Javascript. This method also has its drawbacks, from lack of support for sounds to cross-browser issues, but it is at least accessable to everyone who has a web browser and a text editor. And even if gaming is not Javascript's strong suit, a lot can be learned about the language in the process.

   In this tutorial, we are going to begin making an engine for the creation of games with javascript. Why an engine and not just a game? Because there are a lot of functions that most games share in common and creating an engine will allow us to understand many of the basic concepts behind game design.

   This tutorial aims to fully explain what we are creating and therefore should be easy enough to follow along with, but some amount of prior experience with javascript, html and css will be helpful. If you don't know what a div is or the css background-image property, you probably want to read up on the basics at somewhere like w3schools otherwise, read on.

   As browsers begin to implement the HTML5 specifications the new 'canvas' element is gaining quite a bit of attention, especially for applications like gaming. Using javascript, canvas allows us to create an kind of vector based images imaginable. For this example we are not going to be using canvas, instead focusing on javascript's ability to manipulate an element's css properties, thereby enabling animation. This is not to knock canvas, it has a bright future, however it is still relatively slow and does not render in all browsers (I'm looking at you Internet Explorer).

Beginning

  Before we do anything, you're going to need a few handy tools and a basic file structure. Make sure you have your favorite web browser and text editor ready. Any browser should work, though I reccomend Mozilla Firefox or Google Chrome. For a text editor, you'll want one that at least has syntax highlighting features for html and javascript. If you're a Windows user, I'd suggest Notepad++, if you're on a Mac, TextWrangler is probably your best bet and if you've got a Linux box, you probably don't need me telling you what text editor to use, but you can't go wrong with vim, emacs or Geany (for the non-masochists).

  Now let's create a file structure, assuming you have a folder where you keep programming projects, open it up and create a new folder called 'game' or something of the like. Inside the game folder, create another folder called 'sprites', this is where we will store images for our game. Now open up your text editor and create and save two new files in the game folder named 'index.html' and 'game.js'. 'index.html' is the HTML file that will run our game and 'game.js' is an external javascript where we will create the game. Everything is now ready, so let's dive in!
